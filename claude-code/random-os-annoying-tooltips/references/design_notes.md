# 概要
このSkillは「OSのうざい通知体験」を現代のデスクトップ環境で再現するため設計されました。plyerライブラリを用いてクロスプラットフォームな通知を実現しています。

# 公式ドキュメント抜粋
- plyer: https://plyer.readthedocs.io/en/latest/ 主要OSで同一インターフェースの通知APIを提供。

# 利用例
- 作業中の気分転換・チーム内の小ネタとして。
- コーディング時に「昔のOSのうざさ」を再現して場を和ませる用途。

# 注意点
- 通知頻度はmin/max間隔で調整可能ですが、長時間放置すると煩わしく感じる場合があります。
- plyerの通知はOS側の通知設定に依存するため、表示されない場合はシステム設定を確認してください。

# 設計方針
- 「うざいが笑える」ラインを重視し、実害のない内容のみを採用。
- ログや履歴はセッション内のみ保持し、ファイル保存や外部送信は行いません。
