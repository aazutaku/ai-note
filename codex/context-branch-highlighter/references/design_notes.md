# 概要
context-branch-highlighterは、複数ブランチ・タスクが混在する現場で「今どの作業に集中すべきか」を可視化するためのSkillです。主にGit履歴・ブランチ情報・ファイルの編集時刻をもとに、主要な作業コンテキストを抽出し、迷子や分岐混乱を防ぎます。

# 公式ドキュメント抜粋
- Git公式: https://git-scm.com/doc
- Python os, subprocess: https://docs.python.org/3/library/

# 利用例
- feature/xxx, bugfix/yyyなど複数ブランチで開発中に、直近の編集履歴や現在のHEADブランチをハイライト
- チームでのタスク切り替え時に、どこから再開すべきかを即座に把握

# 注意点
- Git管理下でのみ動作し、巨大バイナリや除外パスは自動無視
- 未コミットの変更も直近編集として扱うため、作業途中でも有効

# 設計方針
CLIサブコマンドでsummary/log/listを切り替え、現場の状況に応じた粒度で情報を提示。既存のGit運用と干渉せず、履歴やファイル情報のみを参照する設計です。
