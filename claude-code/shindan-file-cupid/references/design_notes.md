# 概要
本Skillは、プロジェクト内のファイル名リストをもとに、ファイル同士の“相性”をランダムまたはジョーク的ロジックで診断することを目的としています。実際のファイル内容や依存関係には一切基づかず、完全なアイスブレイク・会話ネタとして設計されています。

# 公式ドキュメント抜粋
- [Python randomモジュール](https://docs.python.org/3/library/random.html) を利用し、診断スコアや理由をランダム生成しています。
- itertools.combinationsにより全ペアを列挙し、最大ペア数を制限しています。

# 利用例
- チームの朝会や雑談タイムに「今日のファイル相性診断」を共有
- 新規プロジェクトのファイル構成紹介時のアイスブレイク
- プロジェクトのREADMEにジョークとして掲載

# 注意点
- 診断は完全なジョークであり、実際の設計・運用判断には使えません。
- ファイルの内容や依存関係を解析しないため、実用的な意味はありません。
- 隠しファイルや一時ファイルも診断対象となる場合があります。

# 設計方針
- ファイルリストのみを入力とし、プロジェクト構成や内容に影響を与えない安全設計
- 出力は会話や雑談に使えるよう、バリエーション豊富な診断理由を用意
- CLIサブコマンドでファイルリストやサマリーも確認可能
