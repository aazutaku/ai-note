# 概要
context-map-mini-viewは、プロジェクトの主要構造とファイル間の関係性を簡潔に可視化するためのスキルです。特に初見や大規模リポジトリでの「迷子防止」と、全体像の即時把握を重視しています。

# 公式ドキュメント抜粋
- Python os.walk: https://docs.python.org/3/library/os.html#os.walk
- Python ast: https://docs.python.org/3/library/ast.html

# 利用例
- 新規参加したプロジェクトで全体の構造を把握したい場合
- 既存のREADMEやPROJECT_MAPが不十分な場合の補助
- 現在作業中のファイルがどのテストや他ファイルに影響を与えるかを確認したい場合

# 注意点
- 除外ディレクトリや.gitignoreに従い、ノイズを最小化していますが、巨大プロジェクトでは要約が粗くなることがあります。
- Python以外の言語依存関係解析は未対応です。

# 設計方針
- 「手軽に・邪魔せず・全体像を一目で」把握できることを優先。
- 既存のドキュメント（README等）があれば積極的に要約に活用。
- ローカルで完結し、プライバシーやセキュリティにも配慮しています。
