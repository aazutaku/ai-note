# 機能概要
`shindan-file-cupid`は、プロジェクト内のファイル名リストを解析し、なぜか“相性の良い同僚ファイル”ペアを診断・推薦するジョーク系Skillです。診断理由は根拠ゼロの謎ロジックやランダム要素で生成され、作業効率には一切寄与しませんが、チームのアイスブレイクや会話のネタとして活用できます。ファイル同士の“恋愛トラブル”が発生するかもしれませんが、実際のファイルやプロジェクトには影響しません。

# 使い方
明示的な呼び出し例：

```
/skill shindan-file-cupid
```

暗黙発動キーワード例：
- 「ファイル同士の相性診断」
- 「同僚ファイルのおすすめペア」
- 「README.mdとLICENSEの関係」

# 出力例
```
main.py ❤️ settings.yaml : 相性度98% - 「設定に敏感な主役、息ピッタリ！」
README.md 💔 LICENSE : 相性度32% - 「お互い真面目すぎて距離感が…」
data.csv 💡 analysis.ipynb : 相性度89% - 「分析されるために生まれたデータ」
test_main.py 🤝 requirements.txt : 相性度54% - 「必要最低限の付き合い」
utils.py 🎲 randomizer.py : 相性度77% - 「気まぐれ同士、意外とウマが合う」
```

# 注意点
- 実際のファイル内容やプロジェクト構成には一切影響を与えません。
- 除外パスやファイル種別の制限はありませんが、隠しファイルや一時ファイルも診断対象になる場合があります。
- 診断結果は完全にジョークであり、作業指針や設計判断には使えません。
- ローカルへのファイル保存や外部API通信は行いません。
