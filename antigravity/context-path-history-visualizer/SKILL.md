# 機能概要
context-path-history-visualizerは、複数のプロンプトやファイル間で移動したコンテキスト（パス）の履歴を自動抽出し、ツリー構造で可視化するSkillです。これにより、現在の作業がどこから始まり、どのような分岐や合流を経て今に至るかを一目で把握できます。検討漏れや再利用すべき情報の発見、思考経路の整理に役立ちます。履歴は自動的に取得され、追加の環境構築や手動保存は不要です。

# 使い方
このSkillは「履歴」「経路」「分岐」「編集フロー」「ツリー表示」などのキーワードが含まれる文脈で自動発動します。明示的な呼び出しは不要ですが、`履歴を可視化して`や`過去のパスを一覧で表示`といった指示でも動作します。

# 出力例
```
root
├── /project/main.py
│   ├── /project/utils/helpers.py
│   │   └── /project/utils/validators.py
│   └── /project/config/settings.yaml
└── /project/README.md
    └── /project/docs/usage.md
```

# 注意点
履歴はセッション内でのみ保持され、ローカルファイルへの永続保存は行いません。一部の一時的なパスや除外ディレクトリ（例: .git, __pycache__）は自動的に除外されます。履歴が途切れている場合や分岐が複雑な場合は、ツリー表示が簡略化されることがあります。
