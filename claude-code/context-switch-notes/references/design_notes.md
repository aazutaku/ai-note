# 概要
context-switch-notesは、複数タスクやプロジェクトを横断する際の「思考の流れ」やTODOの断絶を防ぐために設計されています。各作業コンテキストごとにメモを自動記録・復元し、再開時の知識ロスや手戻りを最小化します。

# 公式ドキュメント抜粋
本Skillは、AIエージェントがプロンプトや作業対象の切り替えを検知したタイミングで自動発動します。CLIから明示的に呼ぶことも可能です。

# 利用例
- プロジェクトAでのバグ修正途中メモを保存し、プロジェクトBに切り替え。
- 再度プロジェクトAに戻った際、直前のメモやTODOが自動で表示され、即座に作業再開。

# 注意点
- メモはローカル保存（notes.json）され、プライバシーが保たれます。
- 冗長なメモ蓄積を防ぐため、各コンテキスト10件まで自動管理。
- 除外パスや一時ファイルは対象外。

# 設計方針
- エージェントのコンテキスト切り替え検知に依存しつつ、CLIからも柔軟に操作可能。
- データ損失を防ぐため、保存前にバックアップを作成。
- シンプルなJSON構造で他ツールとの連携も容易。
