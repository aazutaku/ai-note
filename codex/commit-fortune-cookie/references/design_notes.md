# 概要
commit-fortune-cookieは、Gitコミット時や明示呼び出しで、開発者の作業に“おみくじ”要素を加えるジョーク系スキルです。コミットメッセージやタイムスタンプを乱数シードに使い、毎回異なる運勢・ラッキー言語・アドバイスを生成します。

# 公式ドキュメント抜粋
- Git公式: https://git-scm.com/doc
- Python subprocess: https://docs.python.org/3/library/subprocess.html

# 利用例
- 普段のgit commit時に自動で運勢が表示され、開発の合間の息抜きやチームのネタとして活用できます。
- 明示的に`python commit_fortune_cookie.py show --message "fix: typo"`のように呼び出すことも可能です。

# 注意点
- 出力は標準出力のみ。Git履歴やファイルには影響しません。
- コミットメッセージや時刻が同じ場合、同じ運勢が出る仕様です。
- ローカル環境でのみ動作し、CI/CDやリモート環境では発動しません。

# 設計方針
- シンプルなPythonスクリプトで、既存のワークフローを邪魔しない設計。
- 乱数シードをコミット情報から生成することで、毎回違う結果を担保。
- 拡張性を意識して、ラッキー言語やアドバイスの追加が容易な構造としています。
