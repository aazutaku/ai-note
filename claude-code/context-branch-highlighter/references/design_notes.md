# 概要
context-branch-highlighterは、複数ブランチやタスクが混在する開発現場で、作業者が“今どの流れに注力すべきか”を明示的に提示するためのSkillです。gitリポジトリの履歴やブランチ構造、直近の編集ファイルを解析し、主要な作業コンテキストを抽出します。

# 公式ドキュメント抜粋
- git branch: https://git-scm.com/docs/git-branch
- Python subprocess: https://docs.python.org/3/library/subprocess.html

# 利用例
- 大規模プロジェクトで複数の機能開発やバグ修正が同時進行する場合、作業者が迷子になりにくくなります。
- 直近の編集履歴や分岐状況を一目で把握でき、誤ったブランチでの作業やマージミスを防止します。

# 注意点
- ローカルリポジトリの情報のみを利用し、リモートの状態や未プッシュのブランチは考慮しません。
- 巨大なリポジトリや特殊なブランチ運用（例: rebase中心）では、分岐関係の表示に限界があります。

# 設計方針
- 既存のgitコマンドのみを利用し、外部依存を最小化しています。
- CLI/ターミナル環境での利用を前提とし、出力はシンプルかつ直感的なものとしています。
