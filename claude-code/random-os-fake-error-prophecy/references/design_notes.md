# 概要
`random-os-fake-error-prophecy`は、作業の合間にOSが未来を予言するという非現実的なジョーク通知を生成するSkillです。実際のエラーやシステム挙動には一切影響せず、ユーザーの緊張を和らげるエンタメ用途を主眼としています。

# 公式ドキュメント抜粋
PythonのrandomモジュールやargparseによるCLI設計を採用し、通知内容はテンプレートからランダム生成。ログはホームディレクトリ直下に保存し、過去の予言を参照・集計可能です。

# 利用例
- `/random-os-fake-error-prophecy run` で即座に予言通知
- `build`, `commit`, `deploy` などの操作時に自動発動
- `list`や`summary`サブコマンドで過去の予言を確認

# 注意点
- 本Skillは完全なジョークであり、実際の障害やシステムエラーを模倣・誘発しません
- ログ保存はユーザーのホームディレクトリ配下のみ
- 除外パス指定で重要ディレクトリでは発動しない設計

# 設計方針
通知内容の多様性を重視し、テンプレート・デバイス・イベント・時期を組み合わせて毎回異なる予言を生成。CLIサブコマンドで柔軟な運用が可能です。
