# 概要
このSkillは、コミット前の差分確認を効率化し、主要な変更点や影響範囲の見落としを防ぐことを目的としています。AIによる要約で、レビューやセルフチェックの生産性向上を支援します。

# 公式ドキュメント抜粋
- [git diff](https://git-scm.com/docs/git-diff): 差分取得の標準コマンド
- [OpenAI Chat API](https://platform.openai.com/docs/guides/chat): 差分要約のためのAPI

# 利用例
- コミット前に`python commit_diff_quick_summary.py summary`で要約を取得し、不要な変更や意図しない修正を発見
- 差分ファイル一覧だけを`list`サブコマンドで確認

# 注意点
- 差分が大きすぎる場合は分割要約となり、文脈のつながりが薄れることがあります
- 除外パスはプロジェクト構成により追加調整が推奨されます
- APIキーの管理には十分注意してください

# 設計方針
- 既存のgitワークフローに影響を与えず、ローカルに追加ファイルを残さない
- サブコマンド形式で柔軟に利用可能
- 必要最低限の依存のみで動作
