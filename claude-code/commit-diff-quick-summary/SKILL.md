# 機能概要
このSkillは、コミット前の変更差分（git diff）をAIで自動要約し、主要な変更点や影響範囲を端的にまとめて表示します。手動で差分を読む手間を大幅に削減し、うっかり見落としや不要な変更の混入を防止できます。特に日本語プロジェクトでも高精度な要約が期待でき、既存ワークフローに追加ファイル不要・即導入可能です。履歴やリポジトリ構成への影響もありません。

# 使い方
明示呼び出しは `/commit-diff-quick-summary` で行います。暗黙発動は「この変更の要約」「差分の主な影響」「コミット前に要約したい」といった文脈や、`git diff` の出力が会話に現れた場合に自動で発動します。

# 出力例
```
主要な変更点:
- src/api.py: APIエンドポイントの認証ロジックを追加
- tests/test_api.py: 認証失敗時のテストケースを新規作成
影響範囲:
- API利用時の認証必須化により、既存クライアントの修正が必要
- テストカバレッジ向上
```

# 注意点
- 差分が非常に大きい場合は要約が粗くなることがあります
- .gitignore等で除外されたファイルは対象外です
- ローカルでのみ動作し、履歴やファイルを変更しません
- バイナリや画像の変更は要約されません
