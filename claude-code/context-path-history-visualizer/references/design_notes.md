# 概要
このスキルは、Claude Code のセッション内でのプロンプトやファイル編集の履歴を自動的に記録し、ツリー構造で可視化することを目的としています。履歴はローカルファイルに保存され、セッション全体の思考経路や編集フローを後から俯瞰できます。

# 公式ドキュメント抜粋
Anthropic公式のClaude Code APIは、外部ファイルアクセスや履歴管理の自動化をサポートしていませんが、ユーザーの操作履歴やコンテキスト情報は独自に記録・管理することが推奨されています。

# 利用例
- 複数ファイルを横断した編集経路の可視化
- 過去の分岐点や作業の出発点の特定
- チームでのレビュー時に思考経路を共有

# 注意点
- 履歴はローカルのみに保存され、外部送信はありません
- 除外パスや一時ファイルは自動的に記録対象外となります

# 設計方針
履歴の記録・可視化は単純な時系列だけでなく、分岐や合流を表現できるツリー構造を採用しました。CLIサブコマンドで記録・一覧・可視化・削除を柔軟に操作できます。
