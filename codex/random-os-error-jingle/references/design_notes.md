# 概要
このSkillは、コマンド実行時のエラーを検知し、即座にローカルのジングル音源からランダム選択して再生することで、エラー発生を視覚以外でも直感的に把握できるよう設計されています。音源はユーザーが自由に追加・差し替えでき、通知演出のカスタマイズ性が高い点が特徴です。

# 公式ドキュメント抜粋
- Python subprocess: https://docs.python.org/ja/3/library/subprocess.html
- playsound: https://pypi.org/project/playsound/

# 利用例
- 開発現場でビルドやテスト失敗時に音で即座に把握
- ジョーク用途で作業場の雰囲気作り
- 音源をレトロゲーム風や自作ジングルに差し替えて楽しむ

# 注意点
- サウンド再生はOS依存部分あり。Linux/Mac/Windowsで挙動がやや異なる場合がある
- サーバーや無音環境では効果が限定的
- 音源ファイル管理・著作権に留意

# 設計方針
- サブコマンド(run/list)でCLI操作性を確保
- 音源ディレクトリの柔軟な指定・拡張性
- ミュート・音量調整オプションで実用性も考慮
