# 概要
このSkillは、コミット前の差分要約を自動化することで、開発者の確認作業を効率化します。git diffの出力を解析し、主要な変更点と影響範囲を端的に抽出します。

# 公式ドキュメント抜粋
- [git diff](https://git-scm.com/docs/git-diff): コミット間や作業ツリーの差分を表示する公式コマンド。

# 利用例
- コミット前に`python commit_diff_quick_summary.py summary`を実行し、主要な差分を確認。
- ステージ済み変更のみ要約したい場合は`--cached`を付与。

# 注意点
- 差分が大きすぎる場合やバイナリファイルは要約対象外です。
- 本Skillはローカルでのみ動作し、リポジトリ履歴やファイルに影響を与えません。

# 設計方針
- 既存ワークフローに追加ファイル不要で導入可能。
- 出力は日本語プロジェクトにも配慮。将来的にAI API連携も容易に拡張できる構成です。
