# 機能概要
shindan-file-cupid は、プロジェクト内のファイル名リストをもとに、なぜか「相性がいい同僚ファイル」ペアを診断・推薦するジョーク系Skillです。main.pyとsettings.yamlのような“縁がありそう”なペアや、README.mdとLICENSEのような“微妙な”組み合わせを、根拠ゼロの独自ロジックで出力します。実用性はありませんが、チーム内の会話のネタやアイスブレイクに最適です。

# 使い方
このSkillは明示的な呼び出しは不要です。Antigravityが「ファイル同士の関係性」や「相性診断」「ペア推薦」といったキーワードを検知した際、またはジョーク・アイスブレイク用途が示唆された際に自動発動します。例：
- "このプロジェクトのファイルで相性のいいペアを教えて"
- "ファイル同士の関係性をネタにしたい"

# 出力例
```
[診断結果]
main.py ❤️ settings.yaml
理由: 設定と実装、運命の出会い。
README.md 💔 LICENSE
理由: 価値観の違いで衝突しがち。
data.csv 💞 analysis.ipynb
理由: データと分析、理想的な関係。
.gitignore 🤝 requirements.txt
理由: 地味だけど支え合う仲間。
```

# 注意点
- 診断ロジックは完全にランダムまたは根拠のないもので、実際のプロジェクト運用やファイル内容には一切影響しません。
- 除外パスやシステムファイルは自動的に無視されます。
- 診断履歴はローカルには保存されません。
