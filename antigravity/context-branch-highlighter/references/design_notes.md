# 概要
context-branch-highlighterは、複数ブランチやタスクが並行する現場で「今注力すべき作業流れ」を明示するための設計です。Gitの履歴・ブランチ情報・直近の編集ファイルを統合的に解析し、ユーザーの混乱や誤作業を防ぎます。

# 公式ドキュメント抜粋
- [Git公式](https://git-scm.com/doc): ブランチ管理や履歴取得APIを活用。

# 利用例
- 大規模な開発チームで、複数のfeature/bugfixブランチが同時進行している場合に、主要な作業ブランチや関連ファイルを即座に把握。
- タスク切り替え時に「どこまで進んだか」「どのブランチが主流か」を明確化。

# 注意点
- Git管理外のファイルや履歴は対象外。
- コミット履歴が極端に多い場合、パフォーマンスに影響する可能性があります。

# 設計方針
- 既存のGitコマンドのみを利用し、リポジトリ構造や履歴に変更を加えない安全設計。
- 出力は直感的かつ簡潔にまとめ、混乱を最小限に抑える。
