# 概要
本Skillは、コマンド実行時の失敗体験を和らげ、開発現場の雰囲気を柔らかくするために設計されています。エラー発生時にOS通知領域へ“言い訳”を自動表示し、標準エラー出力も損なわず保持します。

# 公式ドキュメント抜粋
- plyer: https://plyer.readthedocs.io/en/latest/
  - クロスプラットフォームで通知を実現するPythonライブラリです。

# 利用例
- CI/CDパイプラインでコマンド失敗時に通知を受け取る
- 開発チームのペアプロやデバッグ時の気分転換
- ローカル開発環境での自己肯定感アップ

# 注意点
- Linuxでは通知デーモン（例: notify-osd）が必要です。
- plyerは一部OSで通知アイコンやタイムアウトが効かない場合があります。
- ログはユーザーホーム配下に保存されます。

# 設計方針
- OSごとの通知APIはplyerで抽象化
- 言い訳メッセージは毎回ランダム
- エラー内容は通知・標準エラー・ログ全てに残す
