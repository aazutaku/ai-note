# 概要
shindan-file-cupidは、ファイル名リストから完全にジョークベースで『相性のいいファイルペア』を診断・推薦するSkillです。実用性はありませんが、チームのアイスブレイクや雑談のきっかけとして設計されています。

# 設計方針
- ファイル内容や実際の依存関係は一切参照せず、ファイル名だけをもとに診断します。
- 一部のペア（main.pyとsettings.yamlなど）は特別な診断理由を持ちます。
- その他は拡張子や名前の類似度、完全なランダム要素で理由を生成します。
- 除外ファイル（.DS_Storeなど）は自動的に無視されます。

# 利用例
- チームミーティング冒頭のアイスブレイク
- 雑談ネタやペアプロの話題作り
- 「このプロジェクトで一番仲良しなファイルは？」などの遊び用途

# 注意点
- 本Skillは本番環境やファイル内容には一切影響しません。
- 診断結果は完全なジョークであり、実務的な意味はありません。
- 公式APIや外部サービスへの依存はありません。
