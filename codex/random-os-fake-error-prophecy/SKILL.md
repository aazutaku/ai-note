# 機能概要
random-os-fake-error-prophecyは、日々の開発作業中に「未来を予言する」架空のOSエラーメッセージをランダム表示するSkillです。実用性はほぼありませんが、作業のマンネリ化を防ぎ、突然のユーモアや軽い不安を提供します。エディタやターミナルの操作ごとに、まるで本物のエラー通知のような体裁で、根拠のない未来予言が現れます。開発現場の空気を和らげたり、気分転換に役立ちます。

# 使い方
- 明示呼び出し: `/skills random-os-fake-error-prophecy` または `/skills menu` で選択
- 暗黙発動: ターミナルで`git`や`make`等のコマンド実行、エディタでファイル保存やビルド時など、任意の操作を検知して自動発動します。
- 発動例キーワード: "エラー", "警告", "未来", "予言", "失敗"

# 出力例
```
エラー: 明日あなたは3回pushに失敗します
警告: 来週GitHubが逆行します
注意: コードレビューで哲学的質問を受けます
エラー: 2日後、npm installが永遠に終わりません
警告: あなたのCIは満月の夜にだけ動作します
注意: 近日、VSCodeが詩的なメッセージを表示します
```

# 注意点
- 本物のエラー通知と混同しないよう、必ず明確な演出（プレフィックスや色分け等）を行います。
- 実際のシステムやファイルに影響はありません。
- ログや履歴はデフォルトで保存されません（必要に応じて有効化可能）。
- 頻度やタイミングは設定可能ですが、作業を妨げない軽い演出を推奨します。
