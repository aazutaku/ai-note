# 概要
このSkillは、Codexにおける作業履歴や思考経路をツリー構造で記録・可視化し、分岐や合流、編集の流れを直感的に把握できるよう設計されています。履歴はセッションごとに自動抽出され、ユーザーの明示操作や特別な設定なしで利用可能です。

# 公式ドキュメント抜粋
Codex APIの利用にあたっては、OpenAI公式の[APIドキュメント](https://platform.openai.com/docs/)を参照してください。Skill本体は標準的なPythonおよびファイル操作のみで完結します。

# 利用例
- 過去の編集フローを俯瞰し、どこで分岐したかを確認
- 検討漏れや再利用すべき情報の発見
- チーム作業時の経路共有やレビュー

# 注意点
- 一時ファイルや除外パスは自動フィルタされますが、意図しない履歴が混入する場合は手動除外も推奨
- 履歴ファイルはユーザーホームディレクトリに保存されます

# 設計方針
ユーザーの負担を最小限に抑え、履歴管理の透明性と再現性を重視しています。CLIコマンドで柔軟な操作ができ、ツリー表示で現状把握を支援します。
