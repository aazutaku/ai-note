# 概要
このSkillは、開発者がコマンド実行時に遭遇するエラー体験を、ユーモアを交えて前向きに変換することを目的としています。OS通知を活用し、失敗時の心理的負担を軽減します。

# 公式ドキュメント抜粋
- [Python subprocess.run](https://docs.python.org/ja/3/library/subprocess.html): コマンド実行とエラー検出に利用。
- [plyer通知API](https://github.com/kivy/plyer): クロスプラットフォームな通知送信。

# 利用例
- シェルスクリプトや自動化タスクのラップ。
- エラー時の自己肯定・チームの雰囲気改善。

# 注意点
- plyer未導入環境では通知が表示されません（pip install plyerで導入可能）。
- サーバやCUI専用環境では通知が無効化される場合があります。
- 言い訳リストはカスタマイズ可能ですが、Skill本体の編集が必要です。

# 設計方針
- 本来のエラー出力は絶対に失われない設計。
- OS依存を減らすためplyerを採用。
- サブコマンド設計で拡張性を確保。
