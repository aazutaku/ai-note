# 機能概要
context-switch-notesは、複数のタスクやプロジェクトを並行して進める際に、直前の作業内容や思考メモを自動で記録・要約し、コンテキストごとに保存・復元するSkillです。プロンプトや作業対象の切り替えを検知し、そのタイミングで直前の作業要約やTODOリストを追記・格納。次回同じコンテキストに戻った際には、保存されたメモや要約を自動で表示します。これにより、作業の流れや思考を途切れさせず、効率的な再開やタスク管理を支援します。

# 使い方
- 明示呼び出し例: `/skills menu` でSkill一覧から選択、または `$context-switch-notes` のメンションで直接呼び出し。
- 暗黙発動キーワード例: 「プロジェクトAからBに切り替え」「タスク変更」「前回の作業内容を再表示」など、コンテキストやプロンプトの切り替え時に自動発動します。

# 出力例
```
[context-switch-notes] 前回の作業メモ:
- 実装中の関数: fetch_data()
- 未完のTODO: エラーハンドリング追加
- 思考メモ: APIレスポンス形式が不定

[context-switch-notes] 新しいメモを保存しました。
```

# 注意点
- メモはローカルのJSONファイルでコンテキストごとに自動保存されます。
- 大量のメモや冗長な要約を避けるため、保存内容は最新3件まで自動で管理されます。
- データ損失防止のため、保存時にバックアップファイルも生成します。
- 除外パスや一時的な作業領域は自動判別され、記録対象外となります。
