# 概要
本Skillは、スタックトレースから関数・ファイル情報を正確に抽出し、ユーザーの作業コンテキスト（開いているファイルや編集履歴）と突き合わせて、デバッグ時に注目すべき箇所を明示的に提示する設計です。

# 公式ドキュメント抜粋
Python公式traceback: https://docs.python.org/ja/3/library/traceback.html

# 利用例
- 大規模プロジェクトでエラー発生時、どのファイル・関数に注目すべきか即座に特定できます。
- 開いているファイルや直近編集箇所と一致する場合、優先的にハイライトされます。

# 注意点
- スタックトレースの形式が特殊な場合、抽出精度が低下する可能性があります。
- エディタやターミナルの連携状況によっては、開いているファイルや編集履歴が取得できない場合があります。

# 設計方針
- 依存を極力減らし、標準的なPython環境のみで動作可能にしています。
- ファイルや編集履歴の情報は外部保存せず、プライバシーに配慮しています。
