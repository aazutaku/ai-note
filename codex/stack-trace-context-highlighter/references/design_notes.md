# 概要
本Skillは、プログラマがデバッグ時に大量のファイルや関数の中からエラー原因を素早く特定できるよう、スタックトレースと作業コンテキストを自動突き合わせする設計です。

# 公式ドキュメント抜粋
- Python公式Traceback: https://docs.python.org/ja/3/library/traceback.html
- スタックトレースの標準形式を正規表現で解析しています。

# 利用例
- 大規模なPythonプロジェクトで複数ファイルを同時編集している際、エラー発生時にどのファイル・関数に注目すべきか即座に提示。
- エディタやターミナルから環境変数経由で開いているファイル情報を取得することで、ユーザーの作業状況に即したナビゲーションを実現。

# 注意点
- ログが断片的な場合やファイル名が省略されている場合は、推定精度が低下します。
- Python以外の言語スタックトレースは今後拡張予定ですが、現状は一部対応のみ。

# 設計方針
- 環境依存を極力排除し、エディタやターミナルの情報のみで動作。
- ローカルファイルの自動修正や保存は行わず、あくまでナビゲーション支援に特化。
