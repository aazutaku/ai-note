# 概要
本Skillは、現実にはあり得ないユーモラスな“OSパニック”通知をランダムに生成し、作業中の息抜きや気分転換を促す目的で設計されています。通知内容が実際の障害と誤認されないよう、深刻そうで全く深刻でない表現を厳選しています。

# 公式ドキュメント抜粋
通知APIは [plyer](https://plyer.readthedocs.io/en/latest/) を利用し、macOS/Linux/Windowsでクロスプラットフォームに動作します。plyerはPythonからOS標準の通知機能を呼び出せる軽量ライブラリです。

# 利用例
- 集中作業中に突然表示してリフレッシュ効果を狙う
- チーム内の雑談やイベントでネタとして活用
- コードレビューやペアプロの合間のアイスブレイクに

# 注意点
- 通知履歴はローカルのホームディレクトリに保存されます
- 頻度制限(10分間隔)により、作業の妨げになりにくい設計です
- 実際のOS障害通知と混同されないよう、内容・タイトルを工夫しています

# 設計方針
- 通知内容は毎回ランダム
- 履歴管理/頻度制限/CLIサブコマンドにより柔軟な運用が可能
- plyer未導入環境でも標準出力で通知内容を確認できるフェールセーフ
