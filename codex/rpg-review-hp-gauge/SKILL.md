# 機能概要
このSkillは、コードレビューの指摘数を“RPGのHPゲージ”として可視化します。PRごとに指摘数をカウントし、最大HPから減算してHPバーを生成。HPが0になると「力尽きた」演出が表示され、レビューの緊張感や遊び心を演出します。レビューの質や指摘傾向が一目で分かり、開発現場にユーモアと可視化の新しい体験をもたらします。

# 使い方
- 明示呼び出し例: `/skills rpg-review-hp-gauge analyze --pr 1234` または `$rpg-review-hp-gauge @PR-1234`
- 暗黙発動キーワード例: 「HPゲージ」「レビュー指摘数」「体力バー」「コードの体力」などを含む会話やPRコメントで自動発動します。

# 出力例
```
[PR #1234] コードレビューHPゲージ
HP: [█████-----] 50/100
指摘数: 5 (HP -50)
コメント:
- 変数名が不明瞭です
- テストケースが不足しています
...
あなたのコードは半分の体力を失いました！
```
HPが0の場合:
```
HP: [----------] 0/100
あなたのコードは力尽きた...
```

# 注意点
- HPの最大値や指摘ごとの減点値は設定可能
- 指摘内容はGitHub/GitLab APIから取得（APIトークン必須）
- ローカルファイル保存は行いません
- 除外パスやBotコメントは自動で無視します
